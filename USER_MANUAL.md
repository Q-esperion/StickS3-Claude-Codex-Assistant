# StickS3 Claude Codex 小秘书 · 使用说明

基于 M5Stack StickS3 的 Claude / Codex 语音小秘书。屏幕上有 7 个 App：Claude 小秘书、Codex 小秘书、时钟天气仪表盘、红外遥控、网络电台、OTA 无线升级、设置。

---

## 配件

- M5Stack StickS3 主机一台（内置电池 / 屏幕 / 喇叭 / 麦克风 / 红外 / WiFi / IMU）
- USB-C 线（充电 + 首次烧录用）
- 一台常开机的 Windows 电脑（Claude/Codex 小秘书功能要用）

---

## 第一次使用

### 1. 充电开机

USB-C 线插电脑或充电头，充电时屏幕右上角电池图标有闪电。长按左侧电源键开机。

### 2. 配 WiFi

- 如果没连过网，板子会自动变成 WiFi 热点，SSID 叫 `StickS3-Setup`
- 手机（或电脑）连这个热点，**弹出的自动登录页**会跳到配网界面（或手动在浏览器打开 `http://192.168.4.1`）
- 点"配置 WiFi" → 选自己家的 WiFi → 输密码
- **往下拉**还能顺手填几项：
  - **城市代码**：9 位数字，仪表盘天气用；不填则不显示天气。杭州 `101210101`、北京 `101010100`、上海 `101020100`，其他城市自己查 itboy 天气 API 代码表
  - **B 站 UID**：主页 URL 末尾那串数字；留空就不在仪表盘显示粉丝数
  - **科大讯飞语音识别**：填你自己的 IAT 应用 `APPID`、`APISecret`、`APIKey`，Claude/Codex 小秘书语音识别要用。推荐把三项一起粘到"一键粘贴三项"，格式如 `APPID=... APISecret=... APIKey=...`
- 点"保存"，板子自动重启并连上新 WiFi

以后换地方要换 WiFi：进 **设置 → WiFi 配网**，按 B 即可重进配网热点。

### 3. 安装 PC 助手（Claude/Codex 小秘书要用）

在你的 Windows 电脑上：

- 开发者可运行 `python helper\type_server.py`
- 普通用户可从 GitHub Release 下载并双击 `StickS3ClaudeCodexHelper.exe`
- 右下角托盘会出现一个**橙色星星图标**
- 板子通过 UDP 自动发现这台电脑，通常不用手动配

**托盘图标右键菜单**：
- **打开配置** — 改端口、开关 UDP 发现、**设置开机自启**（推荐开）、编辑语音纠错词表
- **绑定 Claude 输入目标 / 绑定 Codex 输入目标** — 3 秒后抓取当前窗口，适合固定粘贴到 VS Code 或终端
- **绑定 Claude 输入框位置 / 绑定 Codex 输入框位置** — 3 秒后抓鼠标所在输入框位置，窗口缩放后会尽量跟随底部输入框
- **打开日志文件夹** — 查看板子和助手的事件日志
- **退出**

PC 不装助手，Claude/Codex 小秘书 App 里会显示"无助手"。

## 按键基本约定

板子只有两个按键：**A**（屏幕左下方）和 **B**（屏幕右下方）。全系统一致：

| 动作 | 效果 |
|---|---|
| **A 短按** | 主动作（菜单切光标 / 电台下一台 / IR 切槽位等）|
| **A 长按** 0.9 秒 | 逆向动作（电台上一台等，看各 App） |
| **B 短按** | 确认 / 进入 / 播放暂停 |
| **B 长按** 0.9 秒 | 返回上一层（从 App 回主菜单；主菜单里长按不处理）|

**屏保**：60 秒无操作自动熄屏，按任意键唤醒。屏幕超时可在设置里改（0/15/30/60/120/300 秒）。

## 各 App 说明

### 1. Claude 小秘书

语音输入工具 + 实时显示 Claude Code 正在做什么。

- **长按 A** 说话，松开 A 触发识别 + 发送
- **识别结果自动粘贴到当前焦点窗口**（推荐终端或编辑器的 Claude Code）
- 识别得不准会在 PC 助手的托盘配置里改纠错词表（免重烧）
- **左侧滚动列表**：Claude Code 的实时动作（工具调用、你的提问、Claude 回复）
  - 青色 = 你的提问
  - 琥珀色 = Claude 回复完成（会响两声"叮咚"提示）
  - 橙色 = 当前正在执行的工具
  - 灰色 = 过往工具
- **短按 B**：2 秒冷静期后触发"测试发送"，会往电脑发一句 `hello from StickS3` 验证链路通不通

想在 PC 端看到 Claude Code 活动要装钩子（项目里的 `.claude/settings.json` 自带），第一次启动 Claude Code 会让你授权 hook，同意即可。

### 2. Codex 小秘书

语音输入到 Codex，并显示 Codex 当前状态。

- **长按 A** 说话，松开 A 触发识别 + 发送
- 识别结果会通过 PC 助手粘贴到 Codex 输入目标
- 托盘里建议分别绑定 **Codex 输入目标** 和 **Codex 输入框位置**，用于 VS Code 内 Codex 面板这类不稳定焦点场景
- 屏幕会显示 Codex 状态：用户提交后「思考中」、调用工具时「运行中」、最终回复后「空闲」
- PC 助手重启后，左侧历史会清空，这是预期行为

### 3. 仪表盘

首页信息聚合：时钟 / 日期星期秒 / 天气 / B 站粉丝数。

- **A 短按**：立即刷新天气和粉丝数（不等后台自动轮询）
- **长按 B**：返回主菜单

天气每 10 分钟自动刷新；B 站粉丝每 2 分钟自动刷新。

### 4. 红外遥控

学习和回放家里遥控器的红外码，4 个槽位持久保存。

- **A 切光标 / B 进详情**
- 在空槽按 B 进入：对准板子按目标遥控器任意键即可学习
- 在已录槽按 B 进入：**再按 B** 发射该码（对着被控设备）
- **长按 B** 返回列表，再次长按 B 回主菜单

**注意**：StickS3 的红外是 IR TX（发射）为主，接收稍弱，学习时把遥控器顶到板子正面。

### 5. 网络电台

在线 MP3 流媒体电台，预置 22 个台（国内 + 国际）。

- **A 短按** = 下一台
- **A 长按** = 上一台
- **B 短按** = 播放/暂停（暂停后再按 B 从当前台继续）
- **长按 B** = 返回主菜单（会记住离开时的台，下次进直接播）
- **进入时自动播放**上次那个台
- 切台时会先停旧台，再连接新台；如果流源卡住，按 B 重试或换台
- 某个台死了（网络断、URL 失效），屏幕提示"流中断 按 B 重试"

**音量**在设置里调（0-10 档）。

**已知不稳的台**：央广中国之声、BBC Radio 1（流源本身就不稳定，死了按 B 换别的台）。

### 6. OTA 升级

无线烧录新固件用，不用拔 USB 线。

- 板子进这个 App → 屏幕会显示板子的 **IP 地址**
- 在 PC 上用 PlatformIO 跑 upload，端口填板子 IP，就能无线刷
- 升级中屏幕显示进度条，100% 后板子自动重启
- **长按 B 退出**（仅在空闲时可退，传输中不可中断）

**适合开发者**。普通用户一般用不到。

### 7. 设置

板子系统参数。按 A 切换光标，超过当前页会自动翻到下一页；按 B 修改值或执行当前动作。

- **音量** 0-10
- **亮度** 1-10
- **自动旋转** 开/关（打开后板子侧置屏幕会翻转 180°）
- **自动熄屏** 关/15秒/30秒/60秒/2分/5分
- **WiFi 配网** 按 B 即刻重进配网热点（换地方/换 WiFi 用）
- **远程 OTA** 按 B 从远程仓库检查固件更新（需要固件预先配置 manifest URL）
- **清除配网** 按 B 清除 WiFi、天气/B 站、讯飞配置并重启

调完**长按 B** 返回会自动保存到 NVS，断电不丢。

## 更新固件（仅开发者）

在 PC 上（需要安装好 Python + PlatformIO）：

1. 板子进 "OTA 升级" App，屏幕记下 IP
2. 在项目目录跑：
   ```
   pio run -t upload
   ```
3. 约 40-65 秒完成，板子自动重启

如果板子 IP 和 `platformio.ini` 里的 `upload_port` 不一致，先改 `platformio.ini`，或临时跑 `python -m platformio run -d . -t upload --upload-port <板子IP>`。

首次烧录必须用 USB 串口（例如 `--upload-port COM6`），以后就能 OTA 了。

远程 OTA 给普通用户用：开发者把 `firmware.bin` 和 `manifest.json` 放到 GitHub Release，固件里配置 `REMOTE_OTA_MANIFEST_URL` 后，用户在 **设置 → 远程 OTA** 里按 B 检查更新。

`manifest.json` 至少包含：

```json
{
  "version": "1.0.0",
  "url": "https://github.com/OWNER/REPO/releases/latest/download/firmware.bin",
  "sha256": "firmware.bin 的 sha256",
  "size": 2957445
}
```

## 常见问题

**Q: 板子听不到声音**
- 设置里看音量是否为 0
- 网络电台首次连可能要 2-3 秒，耐心等
- 充电时可能有轻微电流声，属于硬件特性，拔电或用电池可改善

**Q: "WiFi 未连接" 红字**
- 设置 → WiFi 配网 → 按 B → 手机连 `StickS3-Setup` 重新配

**Q: 仪表盘天气不显示 / 显示的城市不对**
- 设置 → WiFi 配网 → 往下拉改"城市代码"
- 查询城市码：搜"itboy 天气 API 城市代码表"

**Q: B 站粉丝不显示**
- WiFi 配网页里"B 站 UID"留空 = 不显示
- 填错了会显示"B站加载..."，填对刷新几次就有了

**Q: Claude 小秘书里屏幕显示"无助手"**
- 你的电脑上没在运行 `StickS3ClaudeCodexHelper.exe`，去启动
- 确保板子和电脑在同一 WiFi 网段
- UDP 发现失败时板子会用上次缓存的 IP，所以换电脑/换 IP 后可能要重进这个 App 触发一次重新发现

**Q: 录音后提示"请先配讯飞 API"**
- 进 **设置 → WiFi 配网**，按 B 重新打开配网页
- 在页面底部填自己的科大讯飞 IAT 应用 `APPID`、`APISecret`、`APIKey`
- 推荐直接用"一键粘贴三项"，避免把 `APISecret` 和 `APIKey` 填反
- 保存后重启即可；这三个值会写进板子的 NVS，不需要改源码

**Q: 录音后提示"讯飞鉴权失败 401"**
- 这表示三项凭据不匹配，不是 WiFi 或小秘书坏了
- 到科大讯飞控制台确认是同一个 IAT 应用的 `APPID`、`APISecret`、`APIKey`
- 重新进 **设置 → WiFi 配网** 保存一次，优先用"一键粘贴三项"

**Q: 我要开源给别人用，讯飞 API 怎么办**
- 仓库不需要带你的真实 API
- 别人第一次配 WiFi 时，在同一个配网页填自己的讯飞三件套即可
- 开发者如果想编译时预置默认值，可以复制 `src/secrets.example.h` 为 `src/secrets.h`，但 `src/secrets.h` 不会提交到 Git；板子里已经保存的 WiFi 页面配置优先级更高

**Q: 板子连 USB 重启后 COM 口找不到**
- 拔 USB 等 3 秒再插，Windows 会重新枚举。还不行就换 USB 口或换线。

**Q: 板子自动熄屏后按键没反应**
- 按 A 或 B 任意一个都能唤醒屏幕，但**第一次按不触发正常动作**（只唤醒）。再按第二下才生效。

**Q: 电池显示 0% 但板子还在工作**
- M5Unified 的电量估算不准，尤其充满后立刻显示不上去。看着用，没电了会明显卡顿/重启。

## 硬件规格速查

- 主控：ESP32-S3-PICO-1-N8R8（8 MB Flash + 8 MB PSRAM，WiFi + BLE，无蓝牙经典）
- 屏幕：1.14" LCD，135 × 240（可横屏可竖屏）
- 喇叭 / 麦克风：ES8311 codec（I2S）
- 传感器：BMI270 六轴 IMU（重力 / 加速度 / 陀螺仪）
- 红外：TX G46 / RX G42
- 电源：150 mAh 锂电池 + USB-C 充电
- 按键：A (G11) / B (G12) / 电源键
